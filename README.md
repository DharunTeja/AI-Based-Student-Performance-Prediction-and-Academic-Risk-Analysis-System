## 📖 About

Traditional academic systems evaluate students **after** examinations, which delays corrective actions. There is no intelligent system that analyzes historical academic data to predict student performance in advance.

This project addresses that gap by using **AI-driven predictive analytics** to support early academic intervention. Faculty members can input or upload student data and receive **real-time predictions** along with **academic recommendations** generated by an intelligent Risk Advisor Agent.

### 🎯 Project Goals

- Predict student academic performance using machine learning
- Identify academically weak students **before** final examinations
- Reduce manual effort in analyzing student records
- Provide real-time academic insights through an interactive web dashboard
- Assist faculty in making informed, data-driven academic decisions

## 🚀 Getting Started

### Prerequisites

- **Python 3.9** or higher
- **pip** (Python package manager)
- **Git** (optional, for cloning)

### Installation

**1. Clone the repository**

```bash
git clone https://github.com/<your-username>/AI-Student-Performance-Predictor.git
cd AI-Student-Performance-Predictor
```

**2. Create a virtual environment (recommended)**

```bash
# Create virtual environment
python -m venv venv

# Activate it
# On Windows:
venv\Scripts\activate

# On macOS/Linux:
source venv/bin/activate
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Configure environment variables (Optional — for cloud database)**

Create a `.env` file in the root directory (or edit the existing one):

```env
SUPABASE_URL=your_supabase_project_url
SUPABASE_KEY=your_supabase_anon_key
```

> **Note:** If you skip this step, the app will automatically fall back to **local JSON storage**. Everything works without Supabase.

<details>
<summary>📝 <b>Supabase Setup Instructions (Click to expand)</b></summary>

1. Go to [supabase.com](https://supabase.com) and create a free account
2. Create a new project
3. Go to **Settings → API**
4. Copy your **Project URL** → paste as `SUPABASE_URL`
5. Copy the **anon public key** → paste as `SUPABASE_KEY`
6. Go to **SQL Editor** and run the following SQL to create the required tables:

```sql
CREATE TABLE predictions (
    id SERIAL PRIMARY KEY,
    student_data JSONB,
    prediction VARCHAR(10),
    risk_level VARCHAR(20),
    pass_probability FLOAT,
    recommendations_count INT,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE model_metrics (
    id SERIAL PRIMARY KEY,
    model_name VARCHAR(50),
    accuracy FLOAT,
    precision FLOAT,
    recall FLOAT,
    f1_score FLOAT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

</details>

### ▶️ Running the Application

```bash
streamlit run app.py
```

The app will start and open in your browser at **http://localhost:8501**.

## 📋 Usage Guide

Follow these steps to use the system:

### Step 1: Train the Model
Navigate to the **🤖 Model Training** page from the sidebar. Click the train button to train all 3 ML algorithms on the student dataset. The system will automatically select and save the best performing model.

### Step 2: Explore the Dashboard  
Visit the **📊 Dashboard** page to explore interactive data visualizations, distribution charts, correlation analysis, and key academic insights from the student dataset.

### Step 3: Predict Individual Student  
Go to the **🔮 Predict** page, fill in the student's academic attributes (attendance, grades, study time, etc.), and get an instant prediction with:
- **Pass/Fail** classification
- **Risk Level** (High / Medium / Low)
- **Personalized Recommendations** from the AI Risk Advisor

### Step 4: Batch Prediction  
Use the **📁 Batch Upload** page to upload a CSV file containing multiple students' data. The system will process all records and return predictions for the entire class.

### Step 5: View History  
Check the **📜 History** page to review all past predictions, timestamps, risk assessments, and trends over time.

## 🔬 Algorithms

The system uses **Supervised Machine Learning** for binary classification (Pass/Fail).

### Models Implemented

| Algorithm | Description |
|-----------|-------------|
| **Logistic Regression** | Linear model that estimates pass probability using a sigmoid function. Fast and interpretable. |
| **Decision Tree** | Tree-based model that splits data on feature thresholds. Easy to visualize and understand. |
| **Random Forest** | Ensemble of multiple decision trees with majority voting. Generally the most accurate for structured data. |

### ML Pipeline

```
📥 Data Collection → 🧹 Preprocessing → ✂️ Train/Test Split (80/20)
      → 🏋️ Model Training → 📏 Evaluation → 🏆 Best Model Selection → 💾 Save Model
```

### Evaluation Metrics

- **Accuracy** — Overall correct predictions
- **Precision** — Of predicted passes, how many actually passed
- **Recall** — Of actual passes, how many were correctly predicted
- **F1-Score** — Harmonic mean of Precision and Recall
- **Confusion Matrix** — Detailed breakdown of TP, TN, FP, FN

### Risk Classification Thresholds

| Pass Probability | Risk Level | Action |
|:-:|:-:|:--|
| `P(pass) < 0.4` | 🔴 **High Risk** | Immediate intervention needed |
| `0.4 ≤ P(pass) < 0.7` | 🟡 **Medium Risk** | Needs monitoring |
| `P(pass) ≥ 0.7` | 🟢 **Low Risk** | Performing well |

## 📊 Datasets

The project uses the **UCI Student Performance Dataset** as the primary training data:

| Dataset | Records | Features | Description |
|---------|:-------:|:--------:|-------------|
| `student-mat.csv` | 395 | 33 | **Primary** — Math course student data |
| `Student_Performance(1).csv` | 10,000 | 6 | Secondary — Hours studied, scores, activities |
| `StudentsPerformance.csv` | 1,000 | 8 | Tertiary — Demographics & exam scores |

### Key Features (Primary Dataset)

- Demographics: Age, Sex, Address, Family Size
- Academic: Study Time, Failures, Absences, G1, G2 (prior grades)
- Social: Internet Access, Romantic Relationship, Free Time, Going Out
- Support: School Support, Family Support, Extra Paid Classes
- **Target:** G3 (Final Grade, 0–20) → Mapped to **Pass** (≥10) / **Fail** (<10)
